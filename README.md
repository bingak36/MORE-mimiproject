#연습용 웹사이트 크롤링

HTML및 CSS만을 사용하여 최대한 melon 홈페이지와 유사하도록 작성한 html코드입니다.
java script를 사용한 부분은 최대한 제외하고 일부분은 스포티파이의 ui를 가져왔습니다.

---
주제를 멜론 크롤링으로 선정한 이유
---
멜론은 개발자 도구를 켰을 때 주석이 남아있는 보기 좋은 구조의 웹사이트 입니다.
그만큼 초보자가 보기는 편한 부분이 많지만 방식은 매우 구식입니다.
방식이 구식이라는 것은 최신 트랜드에는 못미치더라도 기본기를 닦을 수 있으며 개발자의 관점을 쉽게 바라볼 수 있는 부분에서 특별함을 느꼇습니다.

---
반응형이 없는 이유
---
현재 코드는 반응형이 아닌 어느 특정한 넓이를 정해두고 작성한 코드입니다.
그 이유는 멜론 자체가 고정형 웹사이트이며 모바일은 다른 도메인을 사용하여 유지 보수 되고 있습니다.
네이버와 같은 경우에는 모바일 전용 사이트도 컴퓨터로 접속이 가능하지만
멜론에 같은 경우에는 PC로 접속이 불가 하여 확인하기 어려웠습니다.

---
css 작성시 찾아봤던 점, 신경썻던 점
---
대부분 배웠던 내용을 응용하는 부분이였지만
그 응용을 할 때까지의 발상을 전환하는데 검색이 필요하였습니다.

로그인을 하면 최상단의 유저 네임과 로그 아웃 버튼이 나타난다던가
자바 스크립트 없이도 페이지를 넘겨서 볼 수 있게 한다던가
하는 부분에서 찾아봐야 했던 부분이 있어야 했습니다.

css를 작성할 때 중요하게 생각했던 부분은
기존의 웹사이트의 코드를 보지 않는다였습니다.
html의 틀은 비슷해야 하지만 css를 구현하는 방식에는 다양한 구현 방식이 있습니다.
나만의 구현방식으로 구현을 하면 좀 더 css가 내가 따라하는것이 아니라 내가 사용하는 부분이 된다라는 생각을 가지고 코드를 작성하였습니다. 

---
부족했던 점
---
내가 구현하고 싶은 기능은 java script를 사용해야 하는데
자세히 모를뿐더러 java scrip를 사용하지 않고 구현하고 싶었기 떄문에 기능이 부족한 방식이 되었습니다.

---
마지막으로
---
프로젝트의 주제를 생각하기 까지 오랜 시간이 걸렸습니다.
내가 알고 있는 부분에서 구현할 수 있는 웹사이트는 무엇이 있을까라는 생각이 있었기 떄문에 모자란 부분이 많지만 직접 구현했다는 점에서 만족하고 있습니다.

이후에도 이 웹사이트를 만들게 된다면 반응형 웹사이트로 따로 작성을 하고 싶습니다.
현재로썬 웹사이트의 크기에 따라 변화하는 것이 아무것도 없이 html에서 만들려는 흔적만 남아있습니다.
이 부분이 흔적이 아니라 실제로 작동하는 부분으로 작성하고 싶습니다.
